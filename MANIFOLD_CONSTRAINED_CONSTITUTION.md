# Constitutional Statement of the Manifold-Constrained Method

## Purpose

The manifold-constrained method is a constitutional architecture for global reasoning.
It exists to make closure explicit wherever local power alone is insufficient.

Its purpose is to require that every serious theorem program state, in public form, the rules under which it is allowed to continue, restart, preserve remainder, and identify its endpoint.

## Article I. Admissible Domain

Every theorem program must declare the admissible class on which its claim is made.

Completion requires that the true scope of its operations be stated explicitly.

## Article II. Protected Core

Every theorem program must isolate a projected, protected, or coercive core.

This core is the load-bearing sector on which lawful persistence is tracked.

## Article III. Lawful Continuation

Every theorem program must specify the continuation law by which local control propagates.

Global closure proceeds through an explicit continuation rule.

## Article IV. Restart and Re-entry

Every theorem program must specify the conditions under which restart, normalization, or re-entry is licensed.

Restart is a governed operation.

## Article V. Explicit Remainder

Every theorem program must keep an explicit ledger of what is not yet closed.

Remainder is part of the structure.
Its status is carried explicitly in the public record.

## Article VI. Endpoint Identification

Every theorem program must specify the bridge conditions by which the intended endpoint is identified.

An endpoint is secured under declared criteria of identification.

## Article VII. Final Witness

Every theorem program must provide a final closure witness:

- a strict margin,
- a nondegeneracy bound,
- a rigidity exclusion,
- or an equivalent explicit certificate.

Such a witness marks global completion.

## Constitutional Consequence

The manifold-constrained method holds that a reasoning system reaches maturity when it is rich in local tools and explicit in global closure.

A mature reasoning system can publicly state:

1. where it is allowed to operate,
2. what core it protects,
3. how control propagates,
4. how restart is governed,
5. what remainder remains,
6. how the endpoint is identified,
7. what certifies final closure.

## Canonical Lane

Canonical Lane is the theorem-facing realization of this constitution.

The theorem-library repositories are public demonstrations that this constitution can carry substantial mathematical load across many problem families.
